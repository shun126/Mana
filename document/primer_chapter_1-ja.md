# 第1章 Manaをはじめよう
## Manaとはどんな言語なのか？
Manaとはゲームのイベントを効率良く開発することを目的に開発されたスクリプトです。世の中の物事というものは、常に並行して行われています。
例えば、学校で授業をしている教室においても、教壇で講義する先生、真面目に勉強している生徒、よそ見をしている生徒、無駄話をしている生徒、早弁をしている生徒と、同じ時間に行われている事柄は多種多様です。

しかし、プログラムの処理というものは一つ一つの手順を踏んでいきます。従来のプログラムの手法では、このような並列的な処理というものは困難でした。これを擬似的に再現しようとしたものがManaというスクリプトです。 Manaは疑似並列処理を行うことを目的にしたスクリプトで、複数のアクターという単位によって構成されます。アクターとは日本語で直訳する通りに「役者」の意味であり、そのシーン(シチュエーション)に必要なアクターが、そのシーンのソースファイルの中に存在しています。そして、各アクターがお互いの動きに応じて反応しあい、一つのシーンを作り上げていきます。

さらにわかりやすく言えば、スクリプトを書いている本人は映画でいう監督であり、脚本家であり、演出家であるということになります。そして各アクターはその指示通りに動く俳優なのです。俳優は脚本の内容に従い、お互いのセリフや動きによって、自分のセリフや演技を行います。すべての役者がそれぞれの役割を並行して行う事により、一つのシーンができあがります。
これを可能にしたスクリプトがManaなのです。

## アクターとリクエスト
Manaを象徴する概念が「リクエスト」です。これはアクター同士でお互いの動きを命令しあうもので、これによって並列的な動きとアクター同士の連携を実現しています。例えば、バスに乗っていて“次の停留所が告げられたときに、そこで降りる人が複数いて、同時多発的にボタンを押そうとした”というシチュエーションを考えてみましょう。

まず、「バス」というアクターから「次の停留所で降りるアクターはボタンを押せ」というリクエストが「乗客」アクター全員に発せられます。「乗客」アクターの中で該当するアクターが「ボタンを押せ」というリクエストを受け取って実行します。このとき、該当する「乗客」アクターは他に何かしていても(例えば本を読んでいたり、他の人と話していたり、窓の外を見ていたり)、それらを中断してボタンを押そうとします。つまり「ボタンを押せ」というリクエストはその時にアクターがしている動作よりも優先されます。これが「プライオリティ（優先度）」です。

アクターは特に命令がない限り、他の動作をしています。これは一番低いレベルのリクエストを実行しているのと同じ事です。プライオリティの範囲はより広い整数値を扱えるようになりましたが、メインアクションは従来通りプライオリティ0です。プライオリティ0以下のリクエストは実行されず、プライオリティ1以上のみが実行対象になります。しかし、それよりも優先して行われるべきリクエストが来た場合、それまでしていたことを中断して、そのリクエストを実行しようとします。逆に、あるレベルのリクエストを実行しているときに、それよりも下位のリクエストが来た場合には、現在行われているリクエストが終了するまで、新たに来たリクエストを実行しないで待機させます。

図1-1を見ていただくとリクエストの動作が分かりやすいと思います。実行中のレベルよりもプライオリティの高い要求は即座に実行し、プライオリティの低い要求は終了後に実行されます。

[[Embed(1-1.png, caption=図1-1 プライオリティとリクエストの関係)]]

図1-1は私の週末の生活です。プライオリティ１で私は昼寝をしていましたが、その後プライオリティ３で、テレビを見たい欲求がおこりました。私は食器を洗っていないことを思い出しましたが、プライオリティ２程度の要求だったので、テレビを見た後にしようと思いました。
ところが、プライオリティ４で、妻が私に風呂掃除(とトイレ掃除)を要求してきました。
私はテレビを見ることを中断して掃除を始めます。
掃除終了後、プライオリティ３のテレビを見る処理に戻ります。そして、テレビ番組が終了したので、プライオリティ２の食器洗いを嫌々始めます。

上記のようにアクションが終了すると現在よりも低いプライオリティで登録されたアクションに戻ります。また、今までの処理をキャンセルする事も可能です。会社で居眠りしていたイベントを考えた場合、上司に注意された後再び居眠りするアクションに戻るようでは困る事があります。こういった状況のためにManaではアクション終了時にどのプライオリティまで下がるか指定することが出来ます(rollback命令)。

## Manaでイベントを作るには
では実際にソースを書く場合、どのような書式になるのでしょう。例1-2がManaのスクリプトの表記です。

### 例1-2　初めてのManaソース

```
actor Mr_M
{
    action init
    {
        print(“hello world\n”);
    }
}
```
{～}で囲まれた部分、または;までの行をブロックと呼びます。

actorブロックは先ほど紹介したアクターに当たる部分です。actionブロックはアクターの行動を定義するブロックです。
アクションブロックの名前は予約された名前以外は自由に決めることができます。アクションブロック内に行動を記述することでキャラクターの行動を定義することができます。

では実際に動かすために環境を作ってみましょう。
最近のアプリケーションでは、ウィンドウを開いてそこに文字や絵を表示するような、GUI(グラフィカルユーザインタフェース)を駆使するものが多いようですが、「GUIは使用しているＯＳやツールキットなどに依存するので、汎用のものが作りにくい」という作者の事情があるので、Manaでは謙虚にコンソールを使います。
ここでいうコンソールとは、Windowsの場合はコマンドプロンプト(MS-DOSプロンプト)のことを指します。Linuxなど、Unix系のOSの場合は、ktermやrxvtなどの「ターミナルエミュレータ」と呼ばれるアプリケーションを指します。

[[Embed(1-2.png, caption=図1-2 コマンドプロンプトとターミナルエミュレータの画面例)]]

Windowsを使っている方は、次の手順でコンソールを起動してください。

「スタート」ボタンから「プログラム」→「アクセサリ」→「コマンドプロンプト」
Unixを使っている方は、コンソールに慣れ親しんでいることでしょうから説明を省きます。

早速動かしてみましょう。例1-2のManaプログラムをメモ帳等のエディタを使用して入力して下さい。ファイルにtest.mnという名前で保存したらWindows版はDOSコンソールから、Linux版はターミナルから`mana test.mn`と入力することで実行形式に変換され、Mana実行ファイルがカレントディレクトリに拡張子.mx(今回の場合はtest.mx)として新たに作成されます。この変換作業をコンパイルと呼びます。拡張子の`.mn`はManaプログラムであることを表しています。

秀丸エディタなどキーボードマクロを実行できる環境で、コンパイル操作をキーに割り当てると非常に楽になるので、プログラマに相談してください。

コンパイルが正しく行われても、プログラムが完成したわけではありません。コンパイルされたプログラムは文法的に間違いが無いだけで意図通りに動作するか確認する必要があります。この作業をデバックといいます。デバック作業で問題があったら、エディタでその部分を修正し、サイドコンパイルし、そして動作を確認します。この作業を意図した通りに動くことが確認された時、プログラムは完成したことになります。一連の作業の流れを図示すると以下のようになります。

[[Embed(1-3.png, caption=図1-3 プログラム開発の流れ)]]

mxファイルを実行することでゲームのイベントが開始されるのですが、ゲームタイトル毎に実行方法が違うため本書では実行方法については説明できません。各プロジェクトのプログラマに問い合わせてください。
